# Maestro v1.2 — Hooks Foundation Implementation Plan (Revised)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add hooks-based lifecycle middleware to Maestro for tool permission enforcement, dynamic context injection, and output validation.

**Architecture:** Five hook handlers (SessionStart, BeforeToolSelection, BeforeTool, BeforeAgent, AfterAgent) backed by a compiled permissions manifest. Parallel dispatch enhanced with per-agent environment variables. All 12 agents updated with `model: auto` and canonical tool names.

**Tech Stack:** Shell scripts (bash), JSON, YAML frontmatter, TOML. Python 3 (stdlib only — no third-party packages).

**Design Reference:** `docs/plans/2026-02-17-maestro-v2-hybrid-layered-architecture-design.md`

**Revision Notes (post-review):**
- **REMOVED Task 7 (BeforeModel/thinkingConfig)** — Gemini CLI's `hookTranslator.fromHookLLMRequest()` hard-codes only `temperature`, `maxOutputTokens`, `topP`, `topK`. The `thinkingConfig` field is silently dropped. Thinking level injection is deferred pending upstream fix or SDK migration.
- **INVERTED enforcement hierarchy** — `BeforeTool` (block/deny) is the PRIMARY enforcement gate. `BeforeToolSelection` (union aggregation) is a UX optimization only — it cannot restrict tools, only suggest available ones.
- **ELIMINATED pyyaml dependency** — Permissions generator uses stdlib `re` for YAML frontmatter parsing instead of third-party `yaml` package.
- **FIXED all Python string interpolation** — All `python3 -c` invocations rewritten to use `sys.argv`, `os.environ`, or stdin piping instead of shell variable interpolation into Python source code.
- **FIXED JSON output functions** — All JSON construction uses `python3` for proper escaping instead of string concatenation.
- **ADDED missing tasks** — GEMINI.md update, delegation skill update, validate-agent-permissions.sh update, codebase-investigator resolution.
- **REORDERED tasks** — Output contract unification (Task 10) moved before AfterAgent hook (Task 7) to prevent noisy validation warnings.

**Critical API Constraints:**
- Agent frontmatter uses `.strict()` Zod validation — NO extra fields allowed
- `BeforeTool` hook input does NOT include `agentName` — must track via env var (parallel) or state file (sequential)
- `BeforeToolSelection` uses **union aggregation** — can only ADD tools to the allowed set, never remove them. NOT a security boundary.
- `BeforeTool` uses **OR decision** — any `block`/`deny` wins. This IS the security boundary.
- Sequential `delegate_to_agent` already enforces tool isolation via isolated `ToolRegistry`
- `search_file_content` is a legacy alias for `grep_search`
- `thinkingConfig` in `BeforeModel` hook output is NOT passed through by the translator — deferred
- Hooks communicate via JSON on stdin/stdout. Exit code 0 = success, 1 = non-blocking warning, 2+ = blocking error

```bash
#!/usr/bin/env bash
set -euo pipefail

MAESTRO_HOOKS_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
MAESTRO_ROOT="$(cd "$MAESTRO_HOOKS_DIR/.." && pwd)"

read_stdin() {
  if [ -t 0 ]; then
    echo "{}"
    return
  fi
  cat
}

json_get() {
  local json="$1"
  local key="$2"
  echo "$json" | python3 - "$key" <<'PYEOF' 2>/dev/null || echo ""
import sys, json
d = json.load(sys.stdin)
print(d.get(sys.argv[1], ''))
PYEOF
}

json_get_nested() {
  local json="$1"
  shift
  echo "$json" | python3 - "$@" <<'PYEOF' 2>/dev/null || echo ""
import sys, json
d = json.load(sys.stdin)
for k in sys.argv[1:]:
    if isinstance(d, dict):
        d = d.get(k, '')
    else:
        d = ''
        break
print(d if isinstance(d, str) else json.dumps(d))
PYEOF
}

respond_allow() {
  local msg="${1:-}"
  if [ -n "$msg" ]; then
    python3 - "$msg" <<'PYEOF'
import sys, json
print(json.dumps({"decision": "allow", "systemMessage": sys.argv[1]}))
PYEOF
  else
    echo '{"decision":"allow"}'
  fi
}

respond_block() {
  local reason="$1"
  python3 - "$reason" <<'PYEOF'
import sys, json
print(json.dumps({"decision": "block", "reason": sys.argv[1]}))
PYEOF
}

respond_with_context() {
  local event="$1"
  local context="$2"
  python3 - "$event" "$context" <<'PYEOF'
import sys, json
print(json.dumps({
    "hookSpecificOutput": {
        "hookEventName": sys.argv[1],
        "additionalContext": sys.argv[2]
    }
}))
PYEOF
}

log_hook() {
  local level="$1"
  local msg="$2"
  echo "[$level] maestro: $msg" >&2
}

validate_session_id() {
  local session_id="$1"
  if [[ ! "$session_id" =~ ^[a-zA-Z0-9._-]+$ ]]; then
    log_hook "ERROR" "Invalid session_id: contains unsafe characters"
    return 1
  fi
  return 0
}

get_active_agent() {
  local session_id="$1"
  if ! validate_session_id "$session_id"; then
    echo ""
    return
  fi
  local state_dir="/tmp/maestro-hooks"
  local agent_file="$state_dir/$session_id/active-agent"
  if [ -f "$agent_file" ]; then
    cat "$agent_file"
  else
    echo ""
  fi
}

set_active_agent() {
  local session_id="$1"
  local agent_name="$2"
  if ! validate_session_id "$session_id"; then
    return 1
  fi
  local state_dir="/tmp/maestro-hooks"
  mkdir -p "$state_dir/$session_id"
  local tmp_file="$state_dir/$session_id/active-agent.tmp.$$"
  echo "$agent_name" > "$tmp_file"
  mv "$tmp_file" "$state_dir/$session_id/active-agent"
}

clear_active_agent() {
  local session_id="$1"
  if ! validate_session_id "$session_id"; then
    return 0
  fi
  local state_dir="/tmp/maestro-hooks"
  rm -f "$state_dir/$session_id/active-agent"
}

load_permissions() {
  local perms_file="$MAESTRO_HOOKS_DIR/permissions.json"
  if [ -f "$perms_file" ]; then
    cat "$perms_file"
  else
    echo "{}"
  fi
}

get_agent_tools() {
  local permissions="$1"
  local agent_name="$2"
  echo "$permissions" | python3 - "$agent_name" <<'PYEOF' 2>/dev/null || echo ""
import sys, json
perms = json.load(sys.stdin)
tools = perms.get(sys.argv[1], [])
print(' '.join(tools))
PYEOF
}

is_tool_allowed() {
  local permissions="$1"
  local agent_name="$2"
  local tool_name="$3"
  echo "$permissions" | python3 - "$agent_name" "$tool_name" <<'PYEOF' 2>/dev/null || echo "allowed"
import sys, json
perms = json.load(sys.stdin)
agent_name = sys.argv[1]
tool_name = sys.argv[2]
agent_tools = perms.get(agent_name, None)
if agent_tools is None:
    print('unknown_agent')
elif tool_name in agent_tools:
    print('allowed')
elif tool_name.startswith('maestro__'):
    mcp_base = tool_name[len('maestro__'):]
    if mcp_base in agent_tools or tool_name in agent_tools:
        print('allowed')
    else:
        print('blocked')
else:
    print('blocked')
PYEOF
}
```

**Step 3: Write the permissions generator (stdlib only — no pyyaml)**

Create `hooks/generate-permissions.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

AGENTS_DIR="${AGENTS_DIR:-$(cd "$(dirname "$0")/../agents" && pwd)}"
OUTPUT="${MAESTRO_PERMISSIONS_OUTPUT:-$(cd "$(dirname "$0")" && pwd)/permissions.json}"

if [ ! -d "$AGENTS_DIR" ]; then
  echo "Error: agents directory not found at $AGENTS_DIR" >&2
  exit 1
fi

AGENTS_DIR="$AGENTS_DIR" python3 - "$OUTPUT" <<'PYEOF'
import os, re, json, sys

agents_dir = os.environ['AGENTS_DIR']
output_path = sys.argv[1]
permissions = {}

frontmatter_re = re.compile(r'^---\r?\n(.*?)\r?\n---', re.DOTALL)
name_re = re.compile(r'^name:\s*(.+)$', re.MULTILINE)
tools_re = re.compile(r'^tools:\s*\n((?:\s+-\s+.+\n?)*)', re.MULTILINE)
tool_item_re = re.compile(r'^\s+-\s+(.+)$', re.MULTILINE)

for filename in sorted(os.listdir(agents_dir)):
    if not filename.endswith('.md'):
        continue

    filepath = os.path.join(agents_dir, filename)
    with open(filepath, 'r') as f:
        content = f.read()

    fm_match = frontmatter_re.match(content)
    if not fm_match:
        print(f'Warning: No frontmatter in {filename}', file=sys.stderr)
        continue

    frontmatter = fm_match.group(1)

    name_match = name_re.search(frontmatter)
    if not name_match:
        print(f'Warning: No name in {filename}', file=sys.stderr)
        continue
    name = name_match.group(1).strip()

    tools_match = tools_re.search(frontmatter)
    if not tools_match:
        print(f'Warning: No tools specified for {name}', file=sys.stderr)
        continue

    tools_block = tools_match.group(1)
    tools = [m.group(1).strip() for m in tool_item_re.finditer(tools_block)]

    if not tools:
        print(f'Warning: Empty tools array for {name}', file=sys.stderr)
        continue

    permissions[name] = tools

os.makedirs(os.path.dirname(output_path) or '.', exist_ok=True)
with open(output_path, 'w') as f:
    json.dump(permissions, f, indent=2, sort_keys=True)

print(f'Generated permissions for {len(permissions)} agents -> {output_path}', file=sys.stderr)
PYEOF
```

**Step 4: Write BeforeTool hook (PRIMARY enforcement gate)**

Create `hooks/before-tool.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

HOOK_DIR="$(cd "$(dirname "$0")" && pwd)"
source "$HOOK_DIR/lib/common.sh"

INPUT=$(read_stdin)
SESSION_ID=$(json_get "$INPUT" "session_id")
TOOL_NAME=$(json_get "$INPUT" "tool_name")

AGENT_NAME="${MAESTRO_CURRENT_AGENT:-}"

if [ -z "$AGENT_NAME" ]; then
  AGENT_NAME=$(get_active_agent "$SESSION_ID")
fi

if [ -z "$AGENT_NAME" ]; then
  respond_allow
  exit 0
fi

PERMISSIONS=$(load_permissions)
RESULT=$(is_tool_allowed "$PERMISSIONS" "$AGENT_NAME" "$TOOL_NAME")

case "$RESULT" in
  "allowed")
    respond_allow
    ;;
  "unknown_agent")
    log_hook "WARN" "Unknown agent '$AGENT_NAME' — allowing tool '$TOOL_NAME'"
    respond_allow
    ;;
  "blocked")
```

```bash
run_with_timeout() {
  local timeout_secs="$1"
  shift

  if command -v timeout >/dev/null 2>&1; then
    timeout "$timeout_secs" "$@"
    return $?
  fi

  "$@" &
  local cmd_pid=$!
  local cancel_file="/tmp/maestro-watchdog-$$-$cmd_pid"

  (
    sleep "$timeout_secs"
    if [ ! -f "$cancel_file" ] && kill -0 "$cmd_pid" 2>/dev/null; then
      echo "WARN: Process $cmd_pid timed out after ${timeout_secs}s" >&2
      kill -TERM "$cmd_pid" 2>/dev/null
      sleep 5
      if kill -0 "$cmd_pid" 2>/dev/null; then
        kill -KILL "$cmd_pid" 2>/dev/null
      fi
    fi
  ) &
  local watchdog_pid=$!

  wait "$cmd_pid" 2>/dev/null
  local exit_code=$?

  touch "$cancel_file"
  kill "$watchdog_pid" 2>/dev/null
  wait "$watchdog_pid" 2>/dev/null
  rm -f "$cancel_file"

  return "$exit_code"
}
```
